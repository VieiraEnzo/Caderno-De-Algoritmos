# AGENTS.md

Context file for AI agents working on this repository. This is an **ICPC competitive-programming notebook** ("caderno") for the UFRJ team. It collects C++ algorithm snippets, theory notes, an automated LaTeX build that compiles everything into a printable PDF, and online-judge verification for selected algorithms.

## Algorithm file conventions (`Algoritmos/**/*.h`)

- One algorithm/struct per file. Filenames are PascalCase or lowercase matching the algorithm; extension is `.h`.
- **Line 1** is a `//` comment used as the PDF subsection title, e.g. `//Kosaraju`. Optional flags go at the end in brackets:
  - `[noprint]` excludes the file from the PDF entirely.
  - `[nohash]` skips the content-hash check.
  - Example: `//Title [noprint]`.
- **The first 2 lines are stripped** from the code block that appears in the PDF (see `getlatex.cpp:printa_arquivo_codigo`, `count++ < 2`). Convention: line 1 = title, line 2 = a blank/short `//` line, then description and code from line 3 onward. **Anything on line 2 will NOT appear in the PDF.**
- Description comments (Portuguese or English) and complexity notes (e.g. `//Complexidade: O(n+m)`) are common and appear in the PDF.
- Code is C++17. Snippets generally assume `bits/stdc++.h` is available at the use site and wrap the algorithm in a struct named after it (e.g. `struct Kosa`, `struct Dinic`, `struct SegTree`).
- Files inside a `rem/` subfolder are **completely skipped** by the build — use `rem/` for deprecated, broken, or WIP snippets you want to keep but not publish.

### Example (minimal)

```cpp
//SegTree
//
//Segment tree iterativa para range queries
//Complexidade: O(log n) por query/update
struct SegTree {
    ...
};
```

## Test file conventions (`OJ-tests/**/*.test.cpp`)

- Naming: `<AlgorithmName>.test.cpp` placed in `OJ-tests/<Category>/` matching the algorithm's category folder.
- **First line MUST be** `#define PROBLEM "<judge URL>"` (yosupo, aoj, cses, etc.).
- Then `#include <bits/stdc++.h>`, `using namespace std;`, then `#include "../../Algoritmos/<Category>/<Name>.h"`.
- Implement `int main()` (or `signed main()` when using `#define int long long`) that reads stdin, exercises the algorithm, and writes stdout.
- The CI workflow (`.github/workflows/verify.yml`) runs `oj-verify all --jobs 2 --tle 20 --timeout 3600` on push to master and on PRs. Run locally with `oj-verify run` (see `OJ-tests/localy.txt`).
- Note: some existing tests reference files that were later moved to `rem/` (e.g. `Dsu.test.cpp` includes `Algoritmos/DataStructures/DSU.h`, but `DSU.h` now lives in `rem/DSU.h`). Fix the include path when you touch such a test, or move the file out of `rem/`.

### Example

```cpp
#define PROBLEM "https://judge.yosupo.jp/problem/point_add_range_sum"
#include <bits/stdc++.h>
using namespace std;
#define int long long
#include "../../Algoritmos/DataStructures/BIT.h"

signed main() {
    cin.tie(0);
    ios_base::sync_with_stdio(false);
    int n, q; cin >> n >> q;
    vector<int> a(n);
    for (int i = 0; i < n; ++i) cin >> a[i];
    FenwickTree bit(a);
    while (q--) {
        int t, x, y; cin >> t >> x >> y;
        if (t == 0) bit.add(x, y);
        else cout << bit.sum(x, y - 1) << '\n';
    }
    return 0;
}
```

## Build pipeline (the PDF notebook)

Run from the `Build/` directory:

```sh
./build.sh             # builds ../Pdf/biblioteca.pdf
./build.sh --printall  # also includes files marked [noprint]
```

`build.sh` does:
1. `g++ -std=c++17 -o getlatex getlatex.cpp -O2`
2. `./getlatex [--printall] > biblioteca.tex` — emits the contents of `capa_compact.tex` (preamble + `\begin{document}` + titlepage), then walks `../Algoritmos/`:
   - One `\section{<Category>}` per subdirectory **in directory-iteration order** (NOT alphabetical), skipping `Extra`.
   - Within each category, files are sorted alphabetically by their **title** (the first line, lowercased) and emitted as `\subsection{<title>}` + a `minted{cpp}` block with the first 2 lines stripped.
   - `rem/` subdirectories are skipped entirely.
   - Finally a `\section{Extra}` is emitted; Extra files are printed **verbatim** (no 2-line strip) with the filename as the subsection title, in directory-iteration order.
   - Then `\input{Teoria/teoria.tex}` is emitted (theory chapters).
   - Closes with `\end{multicols}` and `\end{document}`.
3. `rubber -d --shell-escape biblioteca` — compiles with `-shell-escape` (required by `minted`).
4. Moves `biblioteca.pdf` to `../Pdf/` and cleans auxiliary files (`*.aux`, `*.toc`, `*.out`, `*.log`, `*.rubbercache`, `getlatex`).

Dependencies: `g++` (C++17), a LaTeX distribution with `minted`, `extreport`, `multicol`, `fancyhdr`, `titlesec`, `tocloft`, `comment`, `datetime`, `xcolor` (and `[table]`), `hyperref`, `geometry`, `enumitem`, `amsmath`, `amssymb`, `graphicx`; Pygments (for `minted`); and `rubber`.

`Build/biblioteca.tex` and `Build/_minted-biblioteca/` are build artifacts and are gitignored.

## Theory chapters (`Build/Teoria/Capitulos/*.tex`)

- One `.tex` per topic, included by `Teoria/teoria.tex` in this order: `Identidades`, `TeoriaDosNumeros`, `Combinatoria`, `Geometria`, `Grafos`, `Strings`, `Game`, `Other`. (`Shadow.tex` exists but is commented out.)
- Each chapter starts with `\section{<Topic>}` and uses `\subsection{...}` for subtopics.
- Math uses `$$ ... $$` (display) or `$...$` (inline); lists use `\begin{itemize}[leftmargin=*, nosep]`.

## Brute-force stress testing (`setting/bute/`)

- Copy the three templates (`code.cpp`, `brute.cpp`, `gen.cpp`) into a working dir, fill in your solution, a brute-force reference, and a generator that takes a seed from `argv[1]` (see `gen.cpp`'s `rnd`/`uniform_int_distribution` pattern).
- `bash ./bash.sh` compiles all three and loops `gen $i` through seeds 1, 2, 3, ..., feeding each test to both `code` and `brute`; it stops on the first `diff` mismatch and prints the failing case.

## Workflows

### Add a new algorithm

1. Create `Algoritmos/<Category>/<Name>.h`.
2. Line 1: `//<Title>`. Line 2: `//` (blank comment). From line 3: description comments + C++17 code (typically a struct).
3. If the snippet is broken/WIP, put it under `Algoritmos/<Category>/rem/<Name>.h` instead so it stays out of the PDF.
4. (Optional) Add `OJ-tests/<Category>/<Name>.test.cpp` with a `#define PROBLEM` pointing at a judge problem.
5. Rebuild with `cd Build && ./build.sh` to verify it appears in the PDF.

### Add a new theory chapter

1. Create `Build/Teoria/Capitulos/<Topic>.tex` starting with `\section{<Topic>}`.
2. Add `\input{Teoria/Capitulos/<Topic>}` to `Build/Teoria/teoria.tex` in the desired position.
3. Rebuild with `cd Build && ./build.sh`.

## Gotchas

- `getlatex.cpp` skips any directory named `rem`, and the `Extra` category is treated specially (printed verbatim, filename as subsection title, no 2-line strip, not sorted).
- The first 2 lines of every non-Extra `.h` file are stripped from the PDF — keep the title on line 1 and avoid putting meaningful content on line 2.
- `Build/biblioteca.tex` is generated; **do not hand-edit it**. Edit `capa_compact.tex`, `titlepage.tex`, theory chapters, or the algorithm `.h` files instead.
- CI installs `online-judge-verify-helper` from pip and the repo itself (`pip install -U git+https://github.com/<repo>.git@master`). Tests must be discoverable by the verifier (correct `#define PROBLEM` and include paths).
- Author names and date are hardcoded in `Build/titlepage.tex` (currently "Enzo Vieira, Caio David, Luís Rafael", "Setembro, 2025").
- The PDF in `Pdf/biblioteca.pdf` is tracked so it's always available for download without rebuilding.
