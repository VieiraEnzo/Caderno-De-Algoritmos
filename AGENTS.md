# AGENTS.md

Context file for AI agents working on this repository. This is an **ICPC competitive-programming notebook** ("caderno") for the UFRJ team, modeled after [KACTL](https://github.com/kth-competitive-programming/kactl). It collects C++ algorithm snippets and per-chapter theory prose, an automated LaTeX build that compiles everything into a printable PDF, and online-judge verification for selected algorithms.

## Algorithm file conventions (`Algoritmos/**/*.h`)

- One algorithm/struct per file. Filenames are PascalCase or lowercase matching the algorithm; extension is `.h`. **The filename (without `.h`) is the PDF subsection title.**
- Code is C++17. Snippets generally assume `bits/stdc++.h` is available at the use site and wrap the algorithm in a struct named after it (e.g. `struct Kosa`, `struct Dinic`, `struct SegTree`).
- Each file SHOULD begin with a KACTL-style `/** ... */` header documenting the algorithm. **All fields are optional.** Recognized fields:

  | Field | Purpose | Rendered in PDF? |
  | --- | --- | --- |
  | `Author` | Original author credit | No (metadata) |
  | `Date` | When added | No (metadata) |
  | `License` | License (typically CC0) | No (metadata) |
  | `Source` | URL or reference | No (metadata) |
  | `Description` | What it does (multi-line supported) | Yes (as `\textbf{Description:}`) |
  | `Usage` | Example snippet | Yes (as `\textbf{Usage:}` + small minted block) |
  | `Time` | Complexity | Yes (as `\textbf{Time:}`) |
  | `Skip` | `true` to omit from `--skinny` builds (e.g. for skinny WF notebooks) | No (controls inclusion) |
  | `Status` | Testing status (e.g. "tested on yosupo") | No (metadata) |

- **Continuation lines** (not starting with `Field:`) belong to the previous field — use this for multi-line `Description`/`Usage` values.
- The header block is **stripped** from the code block that appears in the PDF; only the code below `*/` is rendered.
- **Legacy files** without the `/** ... */` header are rendered **verbatim** with the filename as the subsection title (the integrator emits a stderr warning). These are being migrated over time — when editing an old file that starts with `//Title`, please convert it to the new header format.
- Files inside a `rem/` subfolder are **completely skipped** by the build — use `rem/` for deprecated, broken, or WIP snippets you want to keep but not publish.

### Example (KACTL-style header)

```cpp
/**
 * Author: Enzo Vieira
 * Date: 2025-09-01
 * License: CC0
 * Source: https://cp-algorithms.com/...
 * Description: Segment tree iterativa para range queries.
 *     Multi-line descriptions are supported; continuation lines
 *     just don't start with "Field:".
 * Usage:
 *     SegTree st(a);
 *     st.update(pos, val);
 *     auto q = st.query(l, r);
 * Time: O(log n) per query/update
 * Skip: true
 * Status: tested on yosupo
 */
struct SegTree {
    ...
};
```

## Chapter prose (`Algoritmos/<cat>/_index.tex`)

- Each category directory has an `_index.tex` (KACTL-style `chapter.tex`) containing the chapter's `\section{<Category>}` followed by theory prose: identities, formulas, notes, tables. There is **no separate theory notebook** — prose is distributed per chapter, alongside the code.
- The integrator inlines `_index.tex` at the top of its category, then emits the category's `.h` files as subsections.
- Math uses `$$ ... $$` (display) or `$...$` (inline); lists use `\begin{itemize}[leftmargin=*, nosep]`.
- To add prose to a chapter, edit the relevant `_index.tex`. To create a new category, make a new directory under `Algoritmos/` with an `_index.tex` starting with `\section{<Name>}`, and add the directory name to `CHAPTERS` in `Build/integrator.py`.

## Test file conventions (`OJ-tests/**/*.test.cpp`)

- Naming: `<AlgorithmName>.test.cpp` placed in `OJ-tests/<Category>/` matching the algorithm's category folder.
- **First line MUST be** `#define PROBLEM "<judge URL>"` (yosupo, aoj, cses, etc.).
- Then `#include <bits/stdc++.h>`, `using namespace std;`, then `#include "../../Algoritmos/<Category>/<Name>.h"`.
- Implement `int main()` (or `signed main()` when using `#define int long long`) that reads stdin, exercises the algorithm, and writes stdout.
- The CI workflow (`.github/workflows/verify.yml`) runs `oj-verify all --jobs 2 --tle 20 --timeout 3600` on push to master and on PRs. Run locally with `oj-verify run` (see `OJ-tests/localy.txt`).
- Note: some existing tests reference files that were later moved to `rem/` (e.g. `Dsu.test.cpp` includes `Algoritmos/DataStructures/DSU.h`, but `DSU.h` now lives in `rem/DSU.h`). Fix the include path when you touch such a test, or move the file out of `rem/`.

### Example

```cpp
#define PROBLEM "https://judge.yosupo.jp/problem/point_add_range_sum"
#include <bits/stdc++.h>
using namespace std;
#define int long long
#include "../../Algoritmos/DataStructures/BIT.h"

signed main() {
    cin.tie(0);
    ios_base::sync_with_stdio(false);
    int n, q; cin >> n >> q;
    vector<int> a(n);
    for (int i = 0; i < n; ++i) cin >> a[i];
    FenwickTree bit(a);
    while (q--) {
        int t, x, y; cin >> t >> x >> y;
        if (t == 0) bit.add(x, y);
        else cout << bit.sum(x, y - 1) << '\n';
    }
    return 0;
}
```

## Build pipeline (the PDF notebook)

Run from the `Build/` directory:

```sh
./build.sh             # builds ../Pdf/biblioteca.pdf (full notebook)
./build.sh --skinny    # omits algorithms whose header has Skip: true (for skinny WF notebooks)
./build.sh --hash      # also emits a 6-char MD5 hash in the corner of each algorithm
```

`build.sh` does:
1. `python3 integrator.py [flags] > biblioteca.tex` — emits the contents of `preamble.tex` (preamble + `\begin{document}` + titlepage + a chapters-only index + `\begin{multicols}{3}`), then walks `../Algoritmos/` **in a fixed chapter order** defined by `CHAPTERS` in `integrator.py` (currently: `Matematica, Geometria, Strings, Grafos, Arvore, DataStructures, DP, Extra`):
   - For each category, inlines `<cat>/_index.tex` if present (it contains `\section{...}` + prose); otherwise emits a bare `\section{<cat>}`.
   - Recursively collects `*.h` from the category (skipping any `rem/` subdirectory), sorts alphabetically by filename, and emits each as `\algotitle{<filename>.h}` (unnumbered, large) + rendered metadata (`\codeheader{Description/Usage/Time}{...}` with bold labels) + a horizontal rule + a `minted{cpp}` block with the header stripped.
   - In `--skinny` mode, files with `Skip: true` are omitted.
   - In `--hash` mode, a 6-char MD5 of the code (whitespace and comments ignored) is emitted right-aligned on the separator rule — useful for verifying you typed an algorithm correctly.
   - Closes with `\end{multicols}` and `\end{document}`.
2. `rubber -d --shell-escape biblioteca` — compiles with `-shell-escape` (required by `minted`).
3. Moves `biblioteca.pdf` to `../Pdf/` and cleans auxiliary files (`*.aux`, `*.toc`, `*.out`, `*.log`, `*.rubbercache`).

### Visual appearance (KACTL-style, defined in `preamble.tex`)

- **Chapter titles** (`\section`) are `\Huge\bfseries` with an underline rule, numbered starting at 1 (`\thesection = \arabic{section}`, no `0.x` prefix).
- **Algorithm titles** use the `\algotitle{<file>.h}` command: the filename **with its `.h` extension**, `\large`, bold, **unnumbered** (they are not `\subsection`s, so they don't appear in the index).
- **Header metadata** is rendered with the `\codeheader{Label}{value}` command (bold label, `\footnotesize`): `\textbf{Description:}`, `\textbf{Usage:}`, `\textbf{Time:}`.
- A **horizontal rule** (`\rule{\linewidth}{0.4pt}`) separates the title/metadata from the code; with `--hash` the 6-char hash sits right-aligned on that line.
- The **index** at the front lists only the major chapters (`\setcounter{tocdepth}{1}`, emitted via `\@starttoc{toc}` inside a 2-column block).

Dependencies: `python3`; a LaTeX distribution with `minted`, `extreport`, `multicol`, `fancyhdr`, `titlesec`, `tocloft`, `titletoc`, `comment`, `datetime`, `xcolor` (and `[table]`), `hyperref`, `geometry`, `enumitem`, `amsmath`, `amssymb`, `graphicx`, `babel`; Pygments (for `minted`); and `rubber`.

`Build/biblioteca.tex` and `Build/_minted-biblioteca/` are build artifacts and are gitignored.

## Brute-force stress testing (`setting/bute/`)

- Copy the three templates (`code.cpp`, `brute.cpp`, `gen.cpp`) into a working dir, fill in your solution, a brute-force reference, and a generator that takes a seed from `argv[1]` (see `gen.cpp`'s `rnd`/`uniform_int_distribution` pattern).
- `bash ./bash.sh` compiles all three and loops `gen $i` through seeds 1, 2, 3, ..., feeding each test to both `code` and `brute`; it stops on the first `diff` mismatch and prints the failing case.

## Common commands

| Task | Command |
| --- | --- |
| Build the notebook PDF (full) | `cd Build && ./build.sh` |
| Build skinny notebook (skips `Skip: true` files) | `cd Build && ./build.sh --skinny` |
| Build with MD5 hashes in corners | `cd Build && ./build.sh --hash` |
| Run integrator only (no LaTeX compile) | `cd Build && python3 integrator.py > biblioteca.tex` |
| Run OJ-verify locally | `cd OJ-tests && oj-verify run` |
| Run CI-style verification | `oj-verify all --jobs 2 --tle 20 --timeout 3600` |
| Stress-test with brute | `cd setting/bute && bash bash.sh` |

## Workflows

### Add a new algorithm

1. Create `Algoritmos/<Category>/<Name>.h`.
2. Start the file with a `/** ... */` header (all fields optional; see the table above). Then the C++17 code (typically a struct). Set `Skip: true` if you want it excluded from skinny WF builds.
3. If the snippet is broken/WIP, put it under `Algoritmos/<Category>/rem/<Name>.h` instead so it stays out of the PDF entirely.
4. (Optional) Add `OJ-tests/<Category>/<Name>.test.cpp` with a `#define PROBLEM` pointing at a judge problem.
5. Rebuild with `cd Build && ./build.sh` to verify it appears in the PDF.

### Add chapter prose / a new theory topic

- To add prose to an existing chapter: edit `Algoritmos/<Category>/_index.tex` (add `\subsection{...}` blocks under the existing `\section{...}`).
- To add a new category: create `Algoritmos/<NewCat>/_index.tex` starting with `\section{<NewCat>}`, add the directory name to `CHAPTERS` in `Build/integrator.py`, and rebuild.

## Gotchas

- The integrator skips any directory named `rem` at any depth. Only `.h` files are treated as algorithms; `_index.tex` is the chapter prose.
- The chapter order in the PDF is fixed by the `CHAPTERS` list in `Build/integrator.py` — it is NOT directory-iteration order. Edit that list to reorder sections.
- `Build/biblioteca.tex` is generated; **do not hand-edit it**. Edit `preamble.tex`, `titlepage.tex`, the per-chapter `_index.tex` files, or the algorithm `.h` files instead.
- CI installs `online-judge-verify-helper` from pip and the repo itself (`pip install -U git+https://github.com/<repo>.git@master`). Tests must be discoverable by the verifier (correct `#define PROBLEM` and include paths).
- Author names and date are hardcoded in `Build/titlepage.tex` (currently "Enzo Vieira, Caio David, Luís Rafael", "Setembro, 2025").
- The PDF in `Pdf/biblioteca.pdf` is tracked so it's always available for download without rebuilding.
- Old `.h` files still using the legacy `//Title` first-line convention (no `/** */` header) are rendered verbatim with the filename as title, and the integrator emits a stderr warning. Migrate them to the new header format when you touch them.
